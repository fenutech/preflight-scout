# Contributing

Preflight Scout is in public alpha. Bug reports, reproducible examples, product
feedback, and security reports are welcome.

## Code and documentation contributions

Preflight Scout uses an AGPL plus commercial dual-licensing model. Preserving a
clear copyright chain is necessary for that model, so unsolicited code and
documentation patches are not accepted during the initial alpha.

Before preparing a pull request:

1. Open an issue describing the problem and proposed change. Do not include
   secrets, confidential source, credentials, or private test evidence.
2. Wait for the maintainer to confirm the scope.
3. For a non-trivial contribution, complete the contributor agreement supplied
   by the maintainer before submitting the change.

The contributor agreement must let the contributor retain ownership while
granting the Preflight Scout copyright owner sufficient copyright and patent
rights to use, modify, sublicense, and relicense the contribution under both
open-source and commercial terms. A Git sign-off, Developer Certificate of
Origin sign-off, or GitHub's default repository contribution terms is not a
substitute for that agreement.

A pull request without the required contributor agreement will not be merged.
An online, lawyer-reviewed CLA workflow may replace this process later.

## Security reports

Do not open a public issue for a suspected vulnerability involving credentials,
authentication state, command execution, or evidence disclosure. Send a
minimal report to [andrea@fenutech.com](mailto:andrea@fenutech.com). Do not send
real credentials, session files, or customer data.
