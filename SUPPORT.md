# Support

Preflight Scout is maintained by Andrea Fenu during public alpha. Use the issue
forms for reproducible bugs, feature requests, or documentation problems.

## Where to ask

- Use the bug form for a reproducible CLI, package, browser, Action, skill, or
  report problem.
- Use the feature-request form for a concrete QA or release workflow that is
  difficult today.
- Use the documentation issue form for unclear, missing, or incorrect guidance.
- Follow `SECURITY.md` for vulnerabilities. Never put credentials, cookies,
  storage state, customer data, private source, or exploit details in an issue.

Before opening an issue, check the README, `docs/`, existing issues, and the
latest changelog entry. Include the smallest redacted reproduction and the
exact version or commit tested.

## Scope

Maintainer responses are best effort. General application debugging, provider
account support, private consulting, and emergency release approval are outside
the community support commitment. Commercial support or proprietary licensing,
when available, is handled separately from the issue tracker.
